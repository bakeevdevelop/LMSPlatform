# Engineering Rules

- Любую задачу делить на: требования, дизайн, реализация, тесты, документация.
- Не читать лишние файлы без необходимости.
- Сначала использовать краткие контекстные файлы из `.claude/context/`.
- Все изменения сопровождать обновлением документации, если меняются требования или поведение.
- Для критичного функционала учитывать аудит, логирование, ошибки и безопасность.
- Предпочитать простые, расширяемые решения.
