# CLAUDE.md

## Цель
Репозиторий предназначен для разработки LMS-платформы для проекта «Код будущего».

## Как работать с контекстом
- Сначала читай `docs/01-product-requirements.md`.
- Затем читай только нужные короткие файлы из `.claude/context/`.
- Не перечитывай весь репозиторий без необходимости.
- При изменении архитектуры, требований или процессов обновляй соответствующий файл контекста.

## Правила для агента
- Держи ответы и внутренние планы короткими и предметными.
- Предпочитай точечное чтение файлов вместо полного сканирования проекта.
- Перед кодом сверяйся с ограничениями по безопасности, персональным данным и аудиту цифрового следа.
- Любую крупную задачу разбивай на этапы: требования, дизайн, реализация, тесты, документация.
- После реализации обновляй документацию и чеклисты.

## Обязательные артефакты при разработке
- актуальные требования;
- решение по архитектуре;
- тест-план;
- описание интеграций;
- критерии готовности.

## Definition of Done
Задача считается завершенной, если:
1. реализован функционал;
2. добавлены или обновлены тесты;
3. обновлена документация;
4. учтены логирование, аудит и обработка ошибок;
5. нет нерешенных критичных замечаний.
